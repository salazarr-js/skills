# salazarr-js/skills

Personal collection of **agents** and **skills** for AI coding assistants.

Compatible with Claude Code and OpenCode.

---

## Install

```bash
npx github:salazarr-js/skills
```

Interactive prompts let you pick agents, skills, platform, and scope:

```
◇  Select agents       — pick which agents to install (space to deselect)
◇  Select skills       — pick which skills to install
◇  Platform            — Claude Code / OpenCode / both
◇  Where to install?   — Global (~/.claude/...) or Project (.claude/...)
```

When both platforms are selected, **skills are installed once under Claude Code** and a symlink is created in the OpenCode path pointing to it — no duplication.

### Paths

| Type | Platform | Global | Project |
|------|----------|--------|---------|
| Agents | Claude Code | `~/.claude/agents/` | `.claude/agents/` |
| Agents | OpenCode | `~/.config/opencode/agents/` | `.opencode/agents/` |
| Skills | Claude Code | `~/.claude/skills/` | `.claude/skills/` |
| Skills | OpenCode | `~/.config/opencode/skills/` | `.opencode/skills/` |

---

## Agents vs Skills

**Agents** run as isolated subagents with their own context window. Spawn them for heavy tasks so they don't pollute your main context.

**Skills** are instructions injected into the current conversation. Use them for recurring workflows that need the full context of what you're working on.

## Adding an agent

1. Copy `agents/example-agent/` → `agents/<your-agent>/`
2. Edit `agent.yaml` — update `description`, `tools`, `model`, and `prompt`
3. Run `pnpm generate` to produce platform files
4. Commit all three: `agent.yaml`, `.claude/agents/<name>.md`, `.opencode/agents/<name>.md`

```
agents/
└── your-agent/
    └── agent.yaml          ← source of truth, edit here
.claude/agents/
└── your-agent.md           ← generated by pnpm generate
.opencode/agents/
└── your-agent.md           ← generated by pnpm generate
```

## Adding a skill

1. Create `skills/<your-skill>/SKILL.md`
2. Add required frontmatter: `name` and `description`
3. Write the instructions below the frontmatter
4. Add to the Skills table above

```
skills/
└── your-skill/
    ├── SKILL.md            ← required
    └── references/         ← optional: reference docs, specs, examples
```
